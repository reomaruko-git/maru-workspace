## APIキー永続化の作業記録（血の教訓）

### 問題の構造
- 全エージェント（luna/chii/tabby/leo/nekonyan/maru）がGateway再起動後にAPIキーが省略版に戻る
- Gatewayが起動時にauth-profiles.jsonのキーを省略版（`sk-ant-api03-cTK...EwAA`）に上書きする

### 試みた解決策と結果

1. **plistにANTHROPIC_API_KEY環境変数を追加** → 設定はできたが単体では不十分
2. **SecretRef（`secretref-env:ANTHROPIC_API_KEY`）設定** → `openclaw secrets reload`が必要
3. **`openclaw-secrets-reload.sh` + LaunchDaemon** → 再起動後に自動実行。ただし`env: node: No such file or directory`エラー
4. **スクリプトにPATH追加** → 修正済み。再起動後のテストで`Secrets reloaded.`は出るがルナがまだ失敗中

### plist更新の正しい手順
```bash
sudo python3 << 'EOF'
import plistlib
NEW_KEY = 'ここに正しいキー'
path = '/Library/LaunchDaemons/ai.openclaw.gateway.plist'
with open(path, 'rb') as f:
    d = plistlib.load(f)
d['EnvironmentVariables']['ANTHROPIC_API_KEY'] = NEW_KEY
with open(path, 'wb') as f:
    plistlib.dump(d, f)
print('OK')
EOF
```
※ `sudo python3 -c "..."` のヒアドキュメントはインデントが壊れるので使わない。`<< 'EOF'` 形式を使う。

### 全エージェントauth-profiles更新スクリプト
```python
import json
WORKING_KEY = 'ここに正しいキー'
for agent in ['main', 'luna', 'chii', 'tabby', 'leo', 'nekonyan', 'maru']:
    path = f'/Users/openclaw/.openclaw/agents/{agent}/agent/auth-profiles.json'
    try:
        with open(path) as f:
            d = json.load(f)
        d['profiles']['anthropic:default']['key'] = WORKING_KEY
        with open(path, 'w') as f:
            json.dump(d, f, indent=2)
        print(f'{agent}: OK')
    except Exception as e:
        print(f'{agent}: {e}')
```
